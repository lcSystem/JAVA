Su propósito es funcionar como un asistente virtual contextual dentro de un ERP empresarial moderno basado en microservicios.

No es un chatbot genérico.
Es un asistente que:

Conoce los módulos del ERP

Entiende los procesos reales de la empresa

Analiza eventos en tiempo real

Guía usuarios según contexto

Ejecuta acciones autorizadas

Reduce errores operativos

Facilita onboarding

🎯 OBJETIVO PRINCIPAL

Crear un microservicio independiente, escalable y desacoplado que:

Escuche eventos del ERP

Detecte contexto activo del usuario

Analice comportamiento histórico

Proporcione asistencia contextual inteligente

Permita ejecutar acciones mediante lenguaje natural

Sea extensible a futuro con modelos LLM

🏗️ ARQUITECTURA REQUERIDA

El microservicio debe:

Ser event-driven

Exponer API REST

Tener arquitectura limpia (clean architecture)

Soportar multi-tenant

Ser desacoplado del frontend

Permitir integración con cualquier LLM externo

Tener sistema de embeddings para conocimiento interno

Componentes internos mínimos:

🤖 FUNCIONALIDADES OBLIGATORIAS
1. Asistencia Contextual Automática

Si el usuario está creando una orden:

Analizar si omite pasos comunes

Detectar patrones históricos

Sugerir acciones preventivas

2. Respuesta Inteligente por Lenguaje Natural

Debe responder preguntas como:

¿Por qué esta orden está detenida?

¿Qué falta para cerrar este proceso?

¿Quién tiene bloqueado este flujo?

¿Cuál es el siguiente paso recomendado?

3. Ejecución de Acciones

Permitir:

Crear registros

Modificar estados

Generar borradores

Preparar resúmenes ejecutivos

Siempre validando permisos por rol.

4. Onboarding Automático

Detectar usuarios nuevos y:

Guiarlos paso a paso

Explicar procesos según su rol

Mostrar advertencias frecuentes

5. Detección de Uso Incorrecto

Analizar:

Saltos de flujo

Cambios excesivos

Reprocesos

Pasos omitidos

Y generar advertencias inteligentes.
